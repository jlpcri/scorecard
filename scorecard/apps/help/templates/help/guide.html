Introduction  

  • About this user guide

    • This document is an introduction to the ScoreCard - A strategy performance management tool.

  • What is ScoreCard?
    • A semi-standard structured report
    • Supported by design methods and automation tools
    • Used by team managers to keep track of execution of activities by the staff within their control
    • Monitor the consequences arising from staff actions

Design  

  • Weekly Data Input

    • The performance data of Person and Team is collected weekly. The entry of data input is added every Friday midnight (include Person and Team). Performance data update need to be finished at the end of next Monday.

  • Data Collection
    • Team member is responsible to him/herself Person performance data input.
    • The performance data of Person will be collected to related Team data.
    • Related data from followed service will be automatically collected to Team Data.
      • jira.west.com
      • isr.west.com
      • West Confluence
      • Pheme
    • Run automation scripts on creation of Person/Team data
      • Filename extension of upload file should be *.py
      • Main function of script should be run_script
      • Callable function should be defined as Global variable inside main function
    • Team Manager is responsible to Team performance data edit.
    • Team member (not manager) cannot access Team Performance Data

Usage  

  • Input Performance Data
    • Each one is responsible for him/herself personal performance data input.
    • Team Manager (include Team Lead) is responsible for team performance data input.
  • Assign User to Team
    • Team Member

      • On first login, each user has the permission to assign him/herself to a team. If the user ignore this option, then him/her should ask his/her supervisor to assign him/her to a team.

    • Team Manager (include team lead)

      • Team manager has the permission to assign team member to your team. And team manager has the permission to assign team member as manager to access team performance data.

  • Export Team Performance Data
    • The format of export file is Excel
    • ScoreCard only export Team Data
    • User can export data per team, per week
    • The data cannot be exported until it is updated by team manager.
  • Number of Phase Lead and Tickets Worked
    • This number is calculated for team Quality Assurance and Test Engineering
    • The number of Phase Lead of person is the counts of phases which lead of that phase is this person.
      • Start delay is the number of business day of difference between the Estimate Start and Actual Start
      • Diff Duration is the number of business days of the difference between the duration of estimate and duration of actual.
    • The number of Tickets Worked of person is the count of tickets which lead is this person
